# 2週間でブラウザシンセを作って収益化した話（1）——コードを書かずに始めた日



大前提としてコードは書けません。今も書けませんし、書いていません。
スタート地点としては、黒い画面（CLI、ターミナルやコマンドラインなど）の操作をやったことがある程度です。

Webサービスの営業をしていた頃、仕事柄「Raspberry Pi（ラズベリーパイ）」という手のひらサイズの小さなコンピュータの存在を知りました。コロナ禍で時間があったので、細かいことは何も考えずに衝動買いして、SSHでいじり倒していました。プログラミングも独学でかじってみましたが、何かを作るところまでは全然たどり着けませんでした。エラーが出た瞬間にお手上げでしたし、考えてみればそもそも私は英語も数学も数字も嫌いでした。

結局辿り着いたのは、黒い画面を見ても怖くはない。で、それで？というだけの状態です。

---

## Claude Codeとの出会い

そのレベルのままWeb業界でスキルを伸ばすこともなく過ごし、昨今の業務ではClaudeのデスクトップ版を好んで使っていたので、流れでClaude Codeも使い始めました。
大量のファイルを読み込んで要約する作業、MDファイルの変換、Claudeに渡すプロンプトの作成——そういう地味だけど時間のかかる作業を任せるレベルで使い、この時点ではまだ「コンピュータ作業ができるClaude」レベルの認識でした。

そんなタイミングで、使っていた音声入力ツールに脆弱性があるというニュースが流れ、代替を探していたところ、「Claude Codeで音声入力アプリ作れた」というXの投稿を見かけました。
マジで？と、とりあえずClaude Codeと話してみたら、

できました。あっさり。

その瞬間に「あ、これは次元が違う」と思いました。と同時に、忘れていたプログラミングで何かを作りたい欲求が沸々と湧き出してきました。
そして私は、仕事中にもかかわらず文鎮になっていた唯一の私用PCであるMacBook Air（2015）を取り出して、勢いそのままにProプランを契約したのです。

---

## 「KAOSS PADを作れるんじゃないか」

このタイミングでは何でもできる、という万能感だけがあって、何を作るかは全く決まっていませんでした。ただ漠然と、作れるかなー、という興味本位の状態です。

そんな時、ちょうどKORGからKAOSS PADの新作が出ました。XYパッドを指でなぞると音が変わる、あの操作感。ずっと好きだったけれど、ちょっと手が出なかった。

そこで、とりあえずClaude Codeに話しかけてみました。

「KAOSS PADみたいなものって作れる？」

期待していた返信は「もちろん！詳しく聞かせてください！」や「それはちょっと難しいですが、XXだったらできますよ！」という『回答』でした。

3分くらい待って出てきた『回答』は、すぐに音ができる楽器の完成品でした。

具体的なアイデアが出てきたのは、その瞬間です。
動くものを目の前に突きつけられた瞬間に、頭の中で次から次へとイメージが巡り始めました。
